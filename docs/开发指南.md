# FishBoss 开发指南

## 1. 环境要求

- Node.js >= 20.0.0
- pnpm >= 9.0.0

## 2. 安装依赖

```bash
# 安装后端依赖
pnpm install

# 安装前端依赖
pnpm --dir frontend install
```

## 3. 环境配置

在项目根目录创建 `.env` 文件：

```env
SERVER_SECRET=your-secret-key
PORT=6577
HOST=0.0.0.0
FISHBOSS_ROOT=~/.fishboss
CORS_ORIGINS=http://localhost:3000,http://localhost:5173,http://localhost:6500
```

## 4. 开发命令

| 命令 | 说明 |
|------|------|
| `pnpm dev` | 启动后端 (端口 6577) |
| `pnpm dev:frontend` | 启动前端 (端口 6500) |
| `pnpm dev:all` | 并行启动前后端 |
| `pnpm build` | 构建后端 |
| `pnpm --dir frontend build` | 构建前端 |
| `pnpm typecheck` | 后端类型检查 |
| `pnpm --dir frontend typecheck` | 前端类型检查 |
| `pnpm test` | 运行测试 |

## 5. 代码规范

### 5.1 后端 (TypeScript)

- 模块结构：**Repository → Service → Routes**
- 每个域有独立文件夹：`modules/<domain>/{repository,service,routes,schema}.ts`
- 使用 **Zod** 做请求/响应 Schema 验证
- 存储路径统一通过 `storage/paths.ts` 获取，禁止硬编码路径

### 5.2 前端 (Vue 3 + TypeScript)

- **组件**：PascalCase 命名，`*.vue` 文件
- **样式**：必须使用 `scoped`，必须使用 CSS 变量
- **图标**：使用 `lucide-vue-next`
- **状态管理**：Pinia Store，按域划分
- **API 调用**：统一通过 `services/` 层
- **i18n**：所有用户可见文本使用 `t('key')`

### 5.3 通用规范

- 禁止添加无意义注释
- 禁止 `as any` / `@ts-ignore`
- 禁止空 `catch` 块
- 必须使用 `console.log` / `console.error` 记录关键操作

## 6. 提交前检查

```bash
# 后端
pnpm typecheck
pnpm build

# 前端
pnpm --dir frontend typecheck
pnpm --dir frontend build

# 测试
pnpm test
```

全部通过方可提交。
