# FishBoss 架构说明

## 1. 架构概述

FishBoss 采用**后端优先**架构：后端处理所有业务逻辑和数据，前端仅负责 UI 展示。

```
┌─────────────────────────────────────────────────────┐
│                   FishBoss                          │
│                                                     │
│  ┌─────────────────┐    ┌────────────────────────┐ │
│  │   Frontend      │───▶│   Backend (Fastify)     │ │
│  │   (Vue 3)       │◀───│   端口 6577             │ │
│  │   端口 6500      │    │                         │ │
│  └─────────────────┘    │  · 所有业务逻辑           │ │
│         Vite Proxy      │  · 所有数据存储            │ │
│         /api/*          │  · AI 编排                 │ │
│                         │  · 工具执行               │ │
│                         │  · SSE 实时通信           │ │
│                         └────────────────────────┘ │
│                                      │              │
│                                      ▼              │
│                              ~/.fishboss/           │
│                              (持久化存储)           │
└─────────────────────────────────────────────────────┘
```

## 2. 前后端职责划分

| 场景 | 处理方 |
|------|--------|
| 业务逻辑 | 后端 |
| 数据存储 | 后端 (~/.fishboss) |
| AI 模型调用 | 后端 |
| 供应商管理 | 后端 |
| 智能体编排 | 后端 |
| 对话管理 | 后端 |
| 工作流执行 | 后端 |
| MCP 服务器管理 | 后端 |
| 技能管理 | 后端 |
| UI 布局/交互 | 前端 |
| 主题切换 | 前端 (后端持久化) |
| 语言切换 | 前端 (后端持久化) |

## 3. 后端结构 (src/)

```
src/
├── bootstrap.ts               # 入口：启动服务器，注册路由
├── index.ts                  # 公共导出
│
├── server/                    # Fastify 服务器
│   ├── bootstrap/             # 实例初始化
│   ├── config/               # 服务器配置
│   ├── domains/              # 路由域 (/api/*)
│   │   ├── conversations/    # 对话管理
│   │   ├── agents/           # 智能体管理
│   │   ├── workflows/        # 工作流
│   │   ├── providers/        # 模型供应商
│   │   ├── mcp.ts           # MCP 服务器
│   │   ├── skills.ts         # 技能
│   │   └── tools/            # 工具注册表
│   ├── middleware/           # 中间件 (认证等)
│   └── sse/                  # SSE 实时通信
│
├── modules/                   # 业务模块
│   ├── auth/                 # 认证
│   ├── conversations/        # 对话
│   ├── agents/               # 智能体
│   ├── workflows/            # 工作流
│   ├── providers/            # 供应商
│   └── mcp/                  # MCP
│
└── storage/                   # 存储路径管理
```

## 4. 前端结构 (frontend/src/)

```
frontend/src/
├── views/                    # 页面视图
├── components/              # Vue 组件
│   └── chat/                # 聊天相关组件
├── stores/                  # Pinia 状态管理
├── services/                # API 服务层
├── composables/             # Vue Composables
├── router/                  # 路由配置
├── i18n/                    # 国际化 (中/英)
└── css/                     # 样式文件
```

## 5. 数据存储 (~/.fishboss/)

```
~/.fishboss/
├── config/                  # 配置文件
├── data/                    # 数据文件
├── auth/sessions/           # Token 会话
├── logs/                    # 运行日志
├── media/                   # 媒体文件
├── prompts/                 # Prompts 文件
├── workflows/               # 工作流数据
├── providers/               # 供应商配置
├── mcp/                     # MCP 服务器配置
├── skills/                  # 技能数据
└── frontend-config/         # 前端配置
```

## 6. API 认证流程

```
前端                          后端
  │                             │
  │  1. 用户输入共享密钥          │
  │────────────────────────────▶│
  │                             │  2. 验证共享密钥
  │                             │  3. 生成 Token (TTL 15min)
  │                             │  4. Session 存储
  │◀────────────────────────────│
  │  返回 Token                   │
  │                             │
  │  后续请求携带 Bearer Token    │
  │────────────────────────────▶│
  │                             │  5. 中间件验证 Token
```
